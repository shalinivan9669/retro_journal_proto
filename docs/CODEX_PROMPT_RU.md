# Промт для Codex

Скопируй это сообщение Codex в корне Godot-проекта.

```text
Ты работаешь в моём Godot 4 проекте. Нужно подготовить игрового монстра Albasty для ретро/low-poly хоррор-уровня.

Контекст существа:
- Албасты появляется редко возле ЛЭП в степи.
- Образ: высокая сгорбленная девушка-йети / степной дух.
- Рост около 2.8 метра.
- Красивая женская нижняя часть лица, но лицо почти закрыто чёрными волосами.
- Волосы очень длинные, чёрные, как ветки/корни, свисают плащом.
- Руки очень длинные, непропорциональные, почти до колен, пальцы костлявые с когтями.
- Одежда/силуэт: куча крупных low-poly треугольников, будто листья/перья/лохмотья.
- Аксессуары: золотая диадема на голове, висюльки с головы, амулеты на груди, красные нитки, бирюзовые бусины, тусклое золото.
- Стиль: clean low-poly, без фотореализма, без шума, крупные читаемые формы, пригодно для Godot.

Что нужно сделать:
1. Проверь наличие Blender в PATH командой `blender --version`.
2. Запусти генератор модели:
   - Windows: `./run_generate_albasty.ps1`
   - либо напрямую: `blender --background --python blender_scripts/create_albasty_lowpoly.py`
3. Убедись, что появился файл `assets/models/albasty_lowpoly.glb`.
4. Если скрипт падает — исправь `blender_scripts/create_albasty_lowpoly.py` и повтори запуск.
5. Проверь, что Godot видит модель по пути `res://assets/models/albasty_lowpoly.glb`.
6. Если в проекте другая структура папок — аккуратно перенеси `assets/models`, `scripts`, `scenes` в правильные места.
7. Подключи сцену `scenes/albasty_instance.tscn` или создай аналогичную сцену, где модель вложена внутрь Node3D с `albasty_controller.gd`.
8. Добавь на уровень несколько Marker3D или Node3D с группой `albasty_spawn_points` возле ЛЭП.
9. Для лошадей добавь группу `horses` на их корневые ноды.
10. Подключи `albasty_spawn_controller.gd` на уровень и назначь `albasty_scene = scenes/albasty_instance.tscn`.
11. Проверь игровой цикл:
    - Албасты появляется возле ЛЭП.
    - Медленно идёт к ближайшей лошади.
    - Если подходит слишком близко, испускает сигнал/можно вызвать панику/провал.
    - Игрок должен не дать ей приблизиться.
12. Не ломай существующую архитектуру. Если есть уже свои классы игрока/лошадей/уровня — интегрируй минимальными изменениями.
13. После изменения проекта дай краткий отчёт: какие файлы созданы/изменены, какие команды запускались, где лежит модель.
```
